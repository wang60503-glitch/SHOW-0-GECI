# SHOW-0 桌面歌词缩放规则说明

整理时间：2026-05-18 16:46

## 1. 当前基础尺寸

- 桌面歌词可见框最大基础尺寸：`506 x 327 px`
- Electron 外层窗口默认尺寸：`514 x 331 px`
- Electron 外层窗口最小尺寸：宽 `508 px`，高 `107 px`
- 第一行图标框长度：`506 px`
- 歌词字号最大值：`96 px`
- 歌词字号最小值：`22 px`

## 2. 改变高度时会怎样

用户把窗口高度缩小时：

- 整个可见歌词框高度会跟着变小。
- 隐藏的歌词显示框也会跟着变矮。
- 第一排歌词仍然贴在歌词显示框的左上角。
- 第二排歌词仍然贴在歌词显示框的右下角。
- 两行歌词中间的空白会优先变小。
- 当高度继续变小时，歌词字号会跟着变小。
- 图标行不会变高、不会变小、不会改变图标位置。
- 图标框仍然保持在上方中心。

当前规则里，隐藏歌词显示框最大状态大概是：

- 顶边：从窗口顶部 `35 px` 开始
- 底边：到窗口顶部约 `307 px`
- 最大高度：约 `272 px`

窗口变矮后：

- 顶边仍然按上方位置开始。
- 底边会跟着窗口底部上移。
- 所以歌词显示框高度会缩小。

## 3. 改变长度时会怎样

这里的“长度”指窗口左右宽度。

设计规则应该是：

- 图标框长度始终保持 `506 px`。
- 图标框始终水平居中。
- 图标里面每个图案的位置不因为窗口变长或变短而改变。
- 歌词显示区域可以跟着窗口宽度变宽或变窄。
- 窗口变长时，单排歌词可显示的字数变多。
- 窗口变短时，单排歌词可显示的字数变少。
- 如果歌词太长，不缩小字体，而是后续按滚动显示处理。

当前最短长度规则：

- `506 px` 是图标框长度，也是歌词界面的最短长度基准。
- 图标框不会因为歌词显示区域变长而变长。

## 4. 哪些位置不变

窗口缩放时，以下内容不应该变：

- 第一行图标框长度不变：`506 px`
- 第一行图标框保持居中
- 图标行高度不变
- 图标内部每个图案的尺寸和相对位置不变
- 第一排歌词始终靠左上
- 第二排歌词始终靠右下
- 唱完/没唱歌词颜色规则不变
- 歌词文字外面的细黑边不变
- 不显示绿色线、红色线、黄色线、蓝色线

## 5. 哪些地方会变

窗口高度变化时，会变化：

- 可见歌词框高度
- 隐藏歌词显示框高度
- 两行歌词中间的空白距离
- 歌词字体大小
- 两行歌词自己的文字字框高度

窗口长度变化时，应该变化：

- 歌词可显示宽度
- 单排能显示的歌词字数
- 歌词过长时的横向显示/滚动策略

窗口长度变化时，不应该变化：

- 图标框长度
- 图标框中心位置
- 图标之间的间距
- 每个图标图案本身的位置

## 6. 字体最大和最小限制

歌词字体最大：

- `96 px`
- 用于正常最大高度状态。

歌词字体最小：

- `22 px`
- 用于窗口高度缩到很矮时。

字体缩放规则：

- 高度够大时，字体保持最大 `96 px`。
- 高度变小时，字体逐步变小。
- 字体最小不能低于 `22 px`。
- 宽度变短时，不用字体缩小解决，后续按滚动显示解决。

## 7. 当前两行歌词位置规则

在隐藏歌词显示框里：

- 第一排歌词：靠左上。
- 第二排歌词：靠右下。
- 中间没有旧的空白行元素。
- 中间空间由布局自动撑开。
- 每一行只保留自己的文字字框，不保留旧的整行大框。

## 8. 如果以后两行歌词改成一行

如果后续把两行歌词改成一行，需要单独确定这一行的位置。

当前不能默认沿用“两行歌词”的位置规则，因为两行规则是：

- 第一排靠左上。
- 第二排靠右下。

改成一行以后，只剩一行歌词，需要重新指定它在隐藏歌词显示框里的位置，比如：

- 靠左上
- 居中
- 靠右下
- 或者其他指定位置

这一条目前只作为后续开发提醒：两行改一行时，必须先确认单行歌词的位置，再改代码。
