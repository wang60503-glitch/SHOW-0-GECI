# 给 v0.app 的使用说明

这是 SHOW-0 桌面歌词界面的完整打包文件。

## 主要看哪个文件

优先看：

`show0-new-ui/desktop-lyrics-project/desktop-lyrics-app.tsx`

这是桌面歌词主界面，包含：

- 图标行
- 歌词框
- 两行歌词布局
- 歌词颜色
- 图标颜色
- 窗口高度缩放规则
- 鼠标拖拽/悬停逻辑

## 页面入口

`show0-new-ui/app/desktop-lyrics-window/page.tsx`

这个文件只负责加载桌面歌词主界面：

```tsx
import { DesktopLyricsProjectApp } from "@/desktop-lyrics-project/desktop-lyrics-app"

export default function DesktopLyricsWindowPage() {
  return <DesktopLyricsProjectApp />
}
```

## 数据和接口

- `show0-new-ui/desktop-lyrics-project/types.ts`
  - 桌面歌词数据结构
  - 当前歌词行、进度、上下两行显示逻辑

- `show0-new-ui/desktop-lyrics-project/show0-bridge.ts`
  - UI 和 Electron/SHOW-0 的接口层

## Electron 窗口相关

- `show0-new-ui/electron/main.cjs`
  - 桌面歌词窗口默认尺寸：`514 x 331`
  - 最小尺寸：`508 x 107`

- `show0-new-ui/electron/preload.cjs`
  - 暴露桌面歌词开关、数据、状态接口

## 本地预览相关

- `show0-new-ui/.codex_tmp/desktop-lyrics-preview-main.cjs`
- `show0-new-ui/.codex_tmp/desktop-lyrics-preview-preload.cjs`

这是本地预览用的 Electron mock，不是正式业务逻辑。

## 不能随便改的规则

- 图标框长度保持 `506 px`
- 图标框保持居中
- 图标框不跟随歌词框缩放
- 可见桌面歌词框最大基础尺寸：`506 x 327 px`
- 最小窗口高度：`107 px`
- 歌词最大字体：`96 px`
- 歌词最小字体：`22 px`
- 第一排歌词固定靠左上
- 第二排歌词固定靠右下
- 不显示绿色线、红色线、黄色线、蓝色线
- 宽度变短时，不靠缩小字体解决，后续应做横向滚动

## 颜色规则

具体看：

`desktop-lyrics-frame-only-20260518_164919/字体颜色和图标颜色.md`

核心颜色：

- 图标默认：`#d8dddd`
- 图标悬停：`#f7ffff`
- 激活蓝色：`#1aa9ff`
- 没唱歌词：`#076ca4 -> #0871aa -> #0978b2`
- 唱完歌词：`#4ffbff -> #ffffff -> #00ffff`

## 歌词框规则

具体看：

`desktop-lyrics-frame-only-20260518_164919/歌词框规则说明.md`

这里写了：

- 高度怎么变
- 长度怎么变
- 哪些位置不变
- 哪些地方会变
- 两行歌词改一行时需要重新确认位置

## v0.app 试做目标

请只围绕桌面歌词 UI 做，不要扩展到酷狗同步、音频播放、VST、歌词解析。

目标是保持当前像素规则和视觉规则，做一个可预览的桌面歌词界面。
